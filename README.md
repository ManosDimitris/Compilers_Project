# Alpha Language Compiler — HY340

> A full compiler and virtual machine for the **Alpha** programming language, built as part of the **HY340 — Compilers & Programming Languages** course at the **Computer Science Department, University of Crete**.

---

## Overview

This project implements a complete compiler pipeline for the **Alpha language** — from raw source code all the way down to executable bytecode running on a custom virtual machine. It is structured in **4 phases**, each building on top of the previous one.

**Languages & Tools used:** C++, Flex (Lex), Bison (Yacc), Makefile

---

## Project Structure

```
Compilers_Project/
├── Phase_1/    # Lexical Analysis (Scanner)
├── Phase_2/    # Syntax Analysis (Parser)
├── Phase_3/    # Semantic Analysis & Intermediate Code Generation
└── Phase_4/    # Target Code Generation & Virtual Machine (AVM)
```

---

## Phases

### Phase 1 — Lexical Analysis
Implements a **scanner** using **Flex** that tokenizes Alpha source code.

- Recognizes all Alpha language tokens: keywords, identifiers, literals, operators, punctuation
- Handles single-line (`//`) and block (`/* ... */`) comments
- Reports lexical errors with line numbers
- Outputs token stream for the parser

**Key files:** `scanner.l`, `Makefile`

---

### Phase 2 — Syntax Analysis
Implements a **parser** using **Bison (YACC)** that defines the grammar of Alpha.

- Full context-free grammar for the Alpha language
- Builds an Abstract Syntax Tree (AST)
- Detects and reports syntax errors
- Integrates with the Phase 1 scanner

**Key files:** `parser.y`, `Makefile`

---

### Phase 3 — Semantic Analysis & Intermediate Code (Quads)
Performs **semantic checks** and generates **intermediate code** in the form of quadruples.

- Symbol table management with nested scope support
- Type checking and semantic validation
- Function definitions and call handling
- Generation of intermediate quadruples (quads) for all language constructs

**Key files:** `*.cpp`, `*.h`, `Makefile`

---

### Phase 4 — Target Code Generation & Alpha Virtual Machine (AVM)
Translates the intermediate quadruples into **binary target code** and executes it on a custom **Alpha Virtual Machine (AVM)**.

- Converts quads to AVM instructions
- Outputs a `.abc` binary file
- Implements the AVM runtime with support for all Alpha operations
- Handles function calls, closures, tables, and built-in library functions

**Key files:** `*.cpp`, `*.h`, `Makefile`

---

##  Build & Run

Each phase has its own `Makefile`. Navigate to the desired phase and build:

```bash
# Example for Phase 4 (full compiler + VM)
cd Phase_4
make

# Run the compiler on an Alpha source file
./acomp <input_file>.asc

# Run the generated bytecode on the AVM
./avm <input_file>.abc
```

---

## Testing

To run the compiler on test files:

```bash
make <input_file>.asc
```

This will compile and execute the specified Alpha source file using the virtual machine.
